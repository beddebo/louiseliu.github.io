h2数据库使用

1 安装

访问http://www.h2database.com/html/download.html，下载最新版h2数据库

配置环境变量

h2-1.3.171.jar  h2.bat  h2.sh  h2w.bat

在不同平台上执行相应脚本，启动H2 gui

2 spring 对H2 数据库的支持

为什么在项目中使用H2数据库

主要是在测试代码中使用，使用内存数据库

配置方式：

两种方式：

1 使用spring xml标签

<jdbc:embedded-database id="dataSource" type="H2">
        <jdbc:script location="classpath:script/schema.sql"/>
    </jdbc:embedded-database>

缺点:

没法指定dataSource的url

2 使用dataSource

    <beans profile="test">
         <context:property-placeholder ignore-resource-not-found="true"
            location="classpath*:/application.properties,
                        classpath*:/application.test.properties" />    


        <bean id="dataSource" class="org.springframework.jdbc.datasource.SimpleDriverDataSource">
            <property name="driverClass" value="${jdbc.driver}" />
            <property name="url" value="${jdbc.url}" />
            <property name="username" value="${jdbc.username}" />
            <property name="password" value="${jdbc.password}" />
        </bean>


        <jdbc:initialize-database data-source="dataSource" ignore-failures="ALL">
            <jdbc:script location="classpath:sql/h2/schema.sql" />
            <jdbc:script location="classpath:data/h2/import-data.sql" encoding="UTF-8"/>
        </jdbc:initialize-database>
    </beans>

spring 测试开发

1 使用spring3.2，shiro作为权限管理的框架，在做junit单元测试时，遇到问题

org.apache.shiro.UnavailableSecurityManagerException: No SecurityManager accessible to the calling code, either bound to the org.apache.shiro.util.ThreadContext or as a vm static singleton.  This is an invalid application configuration.

解决办法： 写一个shiro的test类

在对controller测试时，可以很方便的绑定一个shiro的subject，测试登陆用户和普通用户

2 使用annotaion 方式来ComponentScan

    @ComponentScan(basePackages = {"com.github"}, excludeFilters = {@ComponentScan.Filter(Configuration.class)} )

在进行测试的时候会出现bean还是没有注入的问题。

解决办法：在application-xx.xml文件中配置即有效

<context:component-scan base-package="com.github"></context:component-scan>

猜想 使用java的annotation替换xml中bean的配置，应该和xml 中bean的初始化顺序有关系

git 使用

1. 如何生成key

https://help.github.com/articles/generating-ssh-keys

常用命令：

-1. 下载安装git bash http://code.google.com/p/msysgit/

  1. 设置用户信息： git config --global user.email "xxx" git config --global user.name "xxx"

  2. 检出一个项目： git clone gitxxx

  3. 导入项目到eclipse 2.1 安装EGit

  4. 创建本地branch git branch newfeature

  5. 切换到新创建的branch
    git checkout newfeature

  6. 提交所做修改 5.1 获取修改的内容

     git status

    5.2 添加到commit

     git add file1 file2

    5.3 提交修改

     git commit -m "message"

  7. 回滚到指定版本 6.1 回滚到head

       git reset --hard HEAD

    6.2 回滚到指定版本

       通过git log查询各次提交的版本
   git reset --hard 98507c0010693fa1e122daae5ac32d593258c7ce

    6.3 回滚指定文件

       git checkout HEAD file/to/restore

  8. merge到本地master git checkout master git merge newfeature
